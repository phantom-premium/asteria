# Asteria — мобильное приложение (Android)

## Важно: это не готовый .apk, а проект для сборки

Собрать по-настоящему рабочий, устанавливаемый `.apk` можно только через
Android SDK/Gradle (компиляция ресурсов, dex-байткод, подпись пакета) — этих
инструментов и доступа к интернету для их загрузки в среде, где я работаю,
нет, поэтому готового бинарного файла приложить не могу. Вместо этого — весь
исходный код Android-проекта (Java + Gradle + ресурсы + иконки), который
собирается в APK за пару кликов в Android Studio. Дальше — как это сделать.

## Что это за приложение

Это НЕ отдельная копия мессенджера — сам сайт и база данных как и раньше
запускаются на вашем сервере обычной командой `node server.js` (см. основной
проект). Android-приложение — это лёгкая обёртка (WebView), которая
открывает ваш уже запущенный сервер в полноэкранном окне, как отдельное
приложение с иконкой на рабочем столе телефона, и даёт доступ к
камере/микрофону/загрузке файлов, чтобы звонки, голосовые сообщения и
кружки работали так же, как в браузере.

Адрес вашего сервера (`https://46.8.227.207:3443`) уже зашит в приложение —
при первом запуске оно подключается сразу, ничего вводить не нужно. Сменить
адрес позже (если сервер переедет) можно через меню (⋮ → «Сменить сервер»).

## Собрать APK с телефона, без компьютера (через GitHub Actions)

В проект уже добавлен файл `.github/workflows/build-apk.yml` — это
бесплатный облачный сборщик от GitHub, без рекламы, работает через обычный
браузер (в том числе с телефона). Собирает **release**-сборку (не debug) —
она уже подписана собственным ключом проекта (см. ниже, «Про подпись
release-сборки»), так что готовый файл можно сразу ставить на телефон.

1. Зарегистрируйтесь на [github.com](https://github.com) (бесплатно, если
   аккаунта ещё нет).
2. Создайте новый репозиторий (New repository), например `asteria`.
   **Публичный** — так надёжнее всего работает бесплатный лимит Actions.
3. Загрузите в него содержимое папки `asteria-android/` (именно так, чтобы
   в репозитории появилась папка `asteria-android/`, а не файлы проекта в
   корне — workflow ищет проект по этому пути; если хотите разложить
   иначе, поправьте `working-directory:` в файле workflow).
4. После загрузки сборка запустится автоматически (или вручную: вкладка
   **Actions** → «Build release APK» → **Run workflow**).
5. Дождитесь зелёной галочки (обычно 3–5 минут), откройте этот запуск и
   внизу страницы найдите **Artifacts** → `asteria-release-apk` — это и
   есть готовый APK, скачивается прямо в браузере.
6. Установите скачанный `.apk` на телефон (Android спросит разрешение на
   установку из неизвестных источников — это нормально для приложений не из
   Google Play).

Из других похожих облачных сборщиков без рекламы можно рассмотреть Codemagic
или Bitrise (у обоих есть бесплатный пробный тариф и веб-интерфейс, тоже
открывается с телефона) — но для этого проекта GitHub Actions проще всего,
потому что конфигурация уже готова и ничего дополнительно настраивать не
нужно.

## Про подпись release-сборки

Android не даёт установить *неподписанный* release-APK, поэтому в проект
уже добавлен готовый ключ подписи — `app/keystore/asteria-release.keystore`
(пароль `asteria123`, alias `asteria`, задан прямо в `app/build.gradle`).
Это самоподписанный ключ **для личного использования и тестов**, не для
публикации в Google Play (там нужен отдельный процесс со своим ключом).

⚠️ Если ваш репозиторий на GitHub публичный — этот ключ и пароль от него
тоже виден всем. Для тестовой сборки/личной установки это не проблема
(максимум, что можно сделать с этим ключом — подписать APK с тем же
`applicationId`, который и так не претендует на публикацию). Если в
будущем захотите инвестировать в реальную защиту (например, готовите
приложение к публикации) — сгенерируйте свой ключ и переместите
пароли из `build.gradle` в GitHub Secrets, либо сделайте репозиторий
приватным.

## Как собрать APK (через Android Studio на компьютере)

1. Установите [Android Studio](https://developer.android.com/studio)
   (бесплатно).
2. File → Open → выберите папку `asteria-android` (эту).
3. Android Studio предложит скачать недостающие компоненты (Gradle,
   Android SDK нужной версии) — согласитесь, это займёт несколько минут при
   первом открытии проекта. В проекте специально не приложены сами
   бинарники Gradle wrapper (`gradlew`/`gradle-wrapper.jar`) — Android Studio
   создаёт/чинит их автоматически при открытии; если попросит — нажмите
   «Try Again» / «Sync Now» / предложенный fix.
4. Дождитесь окончания синхронизации (внизу окна, полоса прогресса).
5. Build → Generate Signed Bundle / APK… → APK → выберите существующий
   keystore `app/keystore/asteria-release.keystore` (пароль `asteria123`,
   alias `asteria`, пароль ключа `asteria123`) → release. Либо проще: Build
   → Build App Bundle(s) / APK(s) → Build APK(s) соберёт debug-версию
   (тоже рабочую, просто не release).
6. Готовый файл появится в
   `app/build/outputs/apk/release/app-release.apk` — его можно скопировать
   на телефон и установить (потребуется разрешить установку из неизвестных
   источников), либо сразу нажать «Run ▶» в Android Studio, чтобы
   поставить на подключённый телефон или эмулятор напрямую.

Если у вас уже есть команда/CI с настроенным Android SDK — можно собрать и
без Android Studio: `gradle wrapper && ./gradlew assembleRelease` (первая
команда создаст `gradlew`, если Gradle установлен в системе).

## Про самоподписанный HTTPS-сертификат

Если ваш сервер использует самоподписанный сертификат (тот, что сервер
создаёт автоматически при первом запуске, см. основной README) — при первом
заходе приложение покажет диалог «Сертификат сервера не подтверждён», как
это делает обычный браузер. Это ожидаемо, нажмите «Продолжить». Если у
сервера настоящий сертификат (например, полученный через Let's Encrypt) —
этот диалог не появится вообще.

## Ограничения этой сборки

- Значок приложения — то же изображение, что и иконка в браузерной версии
  (`public/icon-512.png`), пересчитанное под все размеры.
- Push-уведомления (когда приложение свёрнуто) не реализованы — WebView не
  умеет превращать обычные веб-уведомления в системные пуши без отдельного
  сервиса (Firebase Cloud Messaging и т.п.); это отдельная, более крупная
  доработка, если понадобится в будущем.
- Разрешения на камеру/микрофон приложение запрашивает один раз при первом
  запуске — если случайно отклонить, дайте их вручную в системных настройках
  телефона (Настройки → Приложения → Asteria → Разрешения).

## Альтернатива без сборки APK вообще

У сайта уже настроен `manifest.webmanifest` — можно открыть сервер в Chrome
на телефоне и выбрать «Добавить на главный экран» (⋮ → «Установить
приложение» / «Add to Home screen») — получится значок и полноэкранное окно
без установки APK и без Android Studio, прямо сейчас. Разница с настоящим
APK — только в мелочах (например, разрешения на камеру/микрофон запрашивает
сам браузер, а не отдельное приложение).
