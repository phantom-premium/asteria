# Asteria — Android-приложение (WebView-клиент)

## Что это

Это не отдельное приложение с собственной базой данных — это "обёртка" (WebView),
которая открывает ваш уже развёрнутый сервер Asteria (`server.js`) прямо в
Android-приложении. Поскольку сервер отдаёт и сайт, и API из одного процесса с
одной SQLite-базой (`data/asteria.db`), приложение и сайт в браузере автоматически
работают **с одной и той же базой данных, одними и теми же аккаунтами и чатами** —
никакой отдельной синхронизации не требуется.

## Почему нет готового .apk в архиве

Сборка `.apk` требует Android SDK (aapt2, d8, build-tools) и, для самой первой
сборки, доступ в интернет (Gradle скачивает зависимости с google()/mavenCentral()).
В окружении, где я собираю для вас архив, нет ни SDK, ни доступа в сеть — поэтому
я не могу скомпилировать бинарный `.apk` "здесь". Зато весь код полностью готов —
сборка занимает 2–3 минуты в Android Studio.

## Как собрать APK (2 способа)

### Способ 1 — Android Studio (проще всего)
1. Установите [Android Studio](https://developer.android.com/studio).
2. Откройте папку `android/` из этого архива как проект (File → Open).
3. Дождитесь синхронизации Gradle (первый раз скачает зависимости, нужен интернет).
4. Меню **Build → Build App Bundle(s) / APK(s) → Build APK(s)**.
5. Готовый файл появится в `android/app/build/outputs/apk/debug/app-debug.apk`.
6. Перенесите `.apk` на телефон и установите (разрешите установку "из неизвестных источников").

### Способ 2 — командная строка (если уже есть Android SDK)
```bash
cd android
# один раз, если нет gradle-wrapper.jar:
gradle wrapper --gradle-version 8.7
./gradlew assembleDebug
# apk будет в app/build/outputs/apk/debug/app-debug.apk
```

Для релизной (подписанной) версии для публикации используйте
**Build → Generate Signed Bundle / APK** в Android Studio — там же можно создать
новый ключ подписи.

## Как это работает при первом запуске

1. При первом открытии приложение попросит ввести адрес сервера — тот же, что вы
   открываете в браузере (например `https://ваш-домен.ru` или `http://192.168.1.10:3000`).
2. Дальше приложение просто загружает сайт в WebView — все логины, сообщения,
   истории, звонки работают как в браузере, потому что это тот же сервер и та же БД.
3. Адрес сервера можно сменить в любой момент через меню (три точки) → "Сменить сервер".
4. Разрешения на камеру/микрофон запрашиваются автоматически при звонке (WebRTC),
   как и на сайте.

## Важно про сервер
- Сервер должен быть доступен по сети с телефона (белый IP/домен, либо телефон
  в той же локальной сети, что и сервер).
- Если сервер работает по HTTP без сертификата — это уже разрешено в приложении
  (`usesCleartextTraffic="true"`), самоподписанные HTTPS-сертификаты (см.
  `generate-cert.js`) тоже приняты.
- Никаких изменений в `server.js` / базе данных вносить не нужно — сервер как был,
  так и остаётся единой точкой правды и для сайта, и для приложения.
