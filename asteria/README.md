# Asteria — мессенджер для локальной сети

Работает без интернета и без установки внешних пакетов (используются только
встроенные модули Node.js — свой сервер, своя реализация WebSocket и файловая
база данных). Всё, что нужно — это установленный Node.js.

## Новая структура интерфейса

Как в популярных мессенджерах — теперь Настройки и Админ-панель это
полноценные разделы, а не всплывающие окна:

- **На компьютере**: слева узкая рельса с разделами Чаты / Звонки /
  Настройки / Админ (у админов), правее — список чатов и сам чат.
- **На телефоне**: снизу плавающая скруглённая панель Чаты / Звонки /
  Настройки. Админ-панель на телефоне открывается отдельным экраном
  «сверху» (через кнопку в Настройках), а не через нижнюю панель.

## Возможности

- Регистрация / вход, сохранение сессии (cookie на 30 дней)
- Смена логина и пароля, **удаление фото профиля** — всё в разделе Настройки
- **Приватность поиска**: вкладка «Люди» в новом чате больше не показывает
  список всех зарегистрированных пользователей — нужно ввести логин
  (минимум 2 символа), чтобы найти конкретного человека. В Настройках есть
  переключатель «Разрешить находить меня по логину в поиске» — если
  выключить, вас не найдут через поиск вообще (при этом те, у кого уже есть
  с вами чат или общий канал, по-прежнему видят вас как обычно)
- Свой favicon и иконка для установки на главный экран телефона
- Просмотр профиля другого человека (клик по имени собеседника в шапке чата)
- Личные сообщения в реальном времени (WebSocket)
- Каналы: создание, редактирование (название, фото — можно добавить и
  удалить, кто может писать, разрешены ли групповые звонки), удаление —
  доступно владельцу канала (или админу). Подписчики видят статус
  «Вы подписаны» вместо неактивной строки ввода и могут отписаться одной
  кнопкой
- **Групповые звонки в каналах**, где разрешено писать всем подписчикам —
  кнопка 🎧 в шапке чата; владелец может выключить эту возможность в
  настройках канала. Работает как peer-to-peer сетка (каждый соединяется
  с каждым напрямую) — отлично для 2–6 человек, для очень больших групп
  качество может проседать, так как в проекте нет отдельного медиа-сервера
- **Раздел «Звонки»** — история последних звонков (пропущенные, принятые,
  отклонённые, групповые) с возможностью перезвонить одним нажатием
- Реакции на сообщения (эмодзи) — можно ставить, менять и убирать свою реакцию
- Редактирование и удаление собственных сообщений
- Встроенный бот `@asteria_bot` — команды `/help`, `/time`, `/dice`, `/coin`, `/echo текст`
- **Секретная админ-команда**: напишите боту `/openadmin89778958103` — это
  выдаст вашему аккаунту права администратора и откроет раздел «Админ». Там
  можно смотреть список всех пользователей и менять им имя/логин/пароль/
  фото, удалять чужие аккаунты (со всеми их чатами и каналами), смотреть
  список всех чатов и каналов и удалять в них любые сообщения. Кнопка
  «Снять с себя админку» тут же убирает права администратора
- Отправка фото (с предпросмотром и подписью перед отправкой, увеличение по клику),
  видео, файлов, музыки
- Стикеры (эмодзи)
- Голосовые сообщения (запись с микрофона)
- Видео-кружки (запись с камеры, лимит 15 секунд, круглый плеер)
- Истории (фото/видео, живут 24 часа) — свою историю можно удалить в любой момент
- Закрепление важных чатов вверху списка, папки чатов (редактируются в Настройках)
- 6 тем оформления: тёмная, светлая, полночь, закат, лес, роза
- Аудио- и видеозвонки (WebRTC, напрямую между браузерами по локальной сети):
  полноэкранный экран звонка, переключение микрофона/камеры прямо во время
  разговора, переключение фронтальной/задней камеры, **свой видео-квадрат
  можно перетаскивать по экрану и менять местами с видео собеседника**
  одним касанием. «Аудиозвонок» технически всегда включает камеру (сразу
  выключенную), поэтому её можно включить прямо посреди разговора без
  разрывов соединения
- Адаптивный интерфейс — работает и на компьютере, и на телефоне

## Запуск

Требуется Node.js версии **22.5 или выше** (проверить: `node -v`) — из-за
встроенного модуля `node:sqlite`. При запуске в консоли будет строка вида
`ExperimentalWarning: SQLite is an experimental feature` — это нормально,
просто предупреждение Node.js, на работу не влияет.

```bash
cd asteria
node server.js
```

Сервер выведет в консоль список адресов, например:

```
🚀 Asteria запущена!
   Локально:      http://localhost:3000
   В сети (LAN):  http://192.168.1.25:3000
```

- Откройте `http://localhost:3000` на своём компьютере.
- Чтобы к мессенджеру могли подключиться другие устройства в той же Wi-Fi/LAN
  сети — дайте им ссылку вида `http://192.168.1.25:3000` (адрес, который
  сервер напечатал в консоли). Убедитесь, что брандмауэр/файервол вашей ОС
  разрешает входящие подключения на порт 3000.
- Чтобы сервер работал постоянно (не завершался при закрытии терминала),
  запустите его, например, через `nohup node server.js &` (Linux/macOS) или
  как службу/через диспетчер задач на Windows.

Порт можно изменить переменной окружения: `PORT=8080 node server.js`.

## HTTPS (самоподписанный сертификат) — рекомендуется

По умолчанию сервер работает по обычному HTTP. Чтобы получить HTTPS (адресная
строка с замком после подтверждения сертификата, и — что важнее — рабочий
доступ к камере/микрофону на телефонах и других устройствах, заходящих не
через `localhost`), один раз выполните:

```bash
node generate-cert.js
```

Скрипт использует `openssl` (обычно уже установлен на Linux/macOS; на
Windows идёт в комплекте с Git for Windows/Git Bash, либо используйте WSL) и
создаёт сертификат `certs/cert.pem` и `certs/key.pem` с адресами вашего
компьютера в локальной сети.

После этого просто запустите (или перезапустите) сервер как обычно:

```bash
node server.js
```

Он автоматически поднимет HTTPS на порту **3443** (наряду с обычным HTTP на
3000). В консоли появится адрес вида `https://192.168.1.25:3443`.

При первом заходе браузер покажет предупреждение **«Соединение не
защищено»** — это ожидаемо для самоподписанного сертификата (он не подписан
доверенным центром сертификации, а сгенерирован локально). Нажмите
**«Дополнительно» → «Всё равно перейти»** — дальше сайт будет работать как
обычно, просто без зелёной галочки от публичного удостоверяющего центра.

Если позже IP-адрес компьютера в сети изменится и HTTPS перестанет
открываться — удалите папку `certs/` и запустите `node generate-cert.js`
заново.

Порт HTTPS можно поменять переменной окружения: `HTTPS_PORT=8443 node server.js`.

## Данные

- Пользователи, чаты, сообщения и истории хранятся в настоящей базе данных
  **SQLite** — файл `data/asteria.db` (плюс `-wal`/`-shm`, это нормальные
  служебные файлы SQLite при включённом режиме WAL, они не мешают). Модуль
  `node:sqlite` встроен в Node.js начиная с версии 22.5, ставить ничего
  дополнительно не нужно.
- Загруженные файлы (фото, видео, голосовые, кружки, аватары) хранятся в
  папке `uploads/` — это не часть БД, а просто файлы на диске; в базе
  хранится только ссылка на них.
- Чтобы полностью сбросить мессенджер — остановите сервер и удалите
  содержимое папок `data/` и `uploads/`.
- Открыть базу и посмотреть/поправить данные вручную можно любым
  SQLite-клиентом (например, `sqlite3 data/asteria.db` или DB Browser for
  SQLite).

## Добавление своих ботов

Логика бота — в файле `server.js`, функция `botReply(text)`. Чтобы добавить
свою команду, допишите условие в этой функции. Секретная команда для выдачи
прав администратора (`/openadmin89778958103`) обрабатывается отдельно, в
`server.js` она хранится в константе `ADMIN_COMMAND` — рекомендуем сменить
её на что-то своё перед тем, как давать доступ к серверу посторонним.

## Важно про звонки, камеру и микрофон

Аудио/видеозвонки используют технологию WebRTC. Внутри одной локальной сети
они соединяются напрямую между браузерами. Браузеры разрешают доступ к
камере/микрофону только в «безопасном контексте» — то есть по HTTPS или на
`localhost`. Если вы настроили HTTPS (см. раздел выше) — всё работает из
коробки на любом устройстве в сети. Если используете обычный HTTP и заходите
не с самого сервера — Chrome может заблокировать камеру/микрофон; в этом
случае либо настройте HTTPS, либо (временный вариант) откройте
`chrome://flags/#unsafely-treat-insecure-origin-as-secure`, добавьте туда
адрес сервера (`http://192.168.1.25:3000`) и включите флаг.

## Ограничения текущей версии

Это рабочая, но самописная реализация без внешних библиотек — сделана так
специально, чтобы гарантированно запускаться офлайн. Из того, что можно
улучшать дальше: групповые чаты (сейчас только личные и каналы), push-
уведомления, шифрование, более продвинутая модерация каналов, полноценные
пользовательские стикерпаки с картинками вместо эмодзи, групповые звонки.
