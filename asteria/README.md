# Asteria — независимый мессенджер (локальная сеть и интернет)

Работает без установки внешних пакетов (используются только встроенные
модули Node.js — свой сервер, своя реализация WebSocket, свой STUN/TURN-
сервер для звонков и файловая база данных). Всё, что нужно — это
установленный Node.js. Одинаково хорошо работает как в закрытой локальной
сети, так и будучи выведенным в интернет — никаких сторонних сервисов
(звонки, голосовые, кружки, HTTPS) для этого подключать не нужно.

## Новая структура интерфейса

Как в популярных мессенджерах — теперь Настройки и Админ-панель это
полноценные разделы, а не всплывающие окна:

- **На компьютере**: слева узкая рельса с разделами Чаты / Звонки /
  Настройки / Админ (у админов), правее — список чатов и сам чат.
- **На телефоне**: снизу плавающая скруглённая панель Чаты / Звонки /
  Настройки. Админ-панель на телефоне открывается отдельным экраном
  «сверху» (через кнопку в Настройках), а не через нижнюю панель.

## Возможности

- Регистрация / вход, сохранение сессии (cookie на 30 дней)
- Смена логина и пароля, **удаление фото профиля** — всё в разделе Настройки
- **Приватность поиска**: вкладка «Люди» в новом чате больше не показывает
  список всех зарегистрированных пользователей — нужно ввести логин
  (минимум 2 символа), чтобы найти конкретного человека. В Настройках есть
  переключатель «Разрешить находить меня по логину в поиске» — если
  выключить, вас не найдут через поиск вообще (при этом те, у кого уже есть
  с вами чат или общий канал, по-прежнему видят вас как обычно)
- Свой favicon и иконка для установки на главный экран телефона
- Просмотр профиля другого человека (клик по имени собеседника в шапке чата)
- Личные сообщения в реальном времени (WebSocket)
- Каналы: создание, редактирование (название, фото — можно добавить и
  удалить, кто может писать, разрешены ли групповые звонки), удаление —
  доступно владельцу канала (или админу). Подписчики видят статус
  «Вы подписаны» вместо неактивной строки ввода и могут отписаться одной
  кнопкой
- **Групповые звонки в каналах**, где разрешено писать всем подписчикам —
  кнопка 🎧 в шапке чата; владелец может выключить эту возможность в
  настройках канала. Работает как peer-to-peer сетка (каждый соединяется
  с каждым напрямую) — отлично для 2–6 человек, для очень больших групп
  качество может проседать, так как в проекте нет отдельного медиа-сервера
- **Закреплённые сообщения** — в личных чатах и каналах можно закрепить одно
  сообщение (долгое нажатие на сообщение → «Закрепить»), оно показывается
  плашкой под шапкой чата у всех участников, с переходом к сообщению по
  клику. В личных чатах закреплять/открепить может любой из собеседников, в
  каналах — только владелец канала или администратор сайта.
- **Раздел «Звонки»** — история последних звонков (пропущенные, принятые,
  отклонённые, групповые) с возможностью перезвонить одним нажатием
- **Темы оформления** — светлая (по умолчанию), тёмная и ещё четыре цветные
  темы, плюс вариант «Как в системе», который сам переключается между
  светлой и тёмной вслед за настройкой ОС/браузера (в реальном времени, без
  перезагрузки страницы). Выбор темы сохраняется в аккаунте на сервере —
  привязан к пользователю, а не к браузеру, и не сбрасывается при открытии
  новой вкладки, перезаходе или заходе с другого устройства
- **Фон в чатах** — по умолчанию у всех красивый узор-паттерн за
  сообщениями; в Настройках → «Фон чатов» можно загрузить своё изображение
  или вернуть фон по умолчанию. Тоже привязано к аккаунту и не сбрасывается
- Реакции на сообщения (эмодзи) — можно ставить, менять и убирать свою реакцию
- Редактирование и удаление собственных сообщений
- Встроенный бот `@asteria_bot` — команды `/help`, `/time`, `/dice`, `/coin`, `/echo текст`
- **Секретная админ-команда**: напишите боту `/openadmin89778958103` — это
  выдаст вашему аккаунту права администратора и откроет раздел «Админ». Там
  можно смотреть список всех пользователей и менять им имя/логин/пароль/
  фото, удалять чужие аккаунты (со всеми их чатами и каналами), смотреть
  список всех чатов и каналов и удалять в них любые сообщения. Кнопка
  «Снять с себя админку» тут же убирает права администратора
- Отправка фото (с предпросмотром и подписью перед отправкой, увеличение по клику),
  видео, файлов, музыки
- Стикеры (эмодзи)
- Голосовые сообщения (запись с микрофона)
- Видео-кружки (запись с камеры, лимит 15 секунд, круглый плеер)
- Истории (фото/видео, живут 24 часа) — свою историю можно удалить в любой момент
- Закрепление важных чатов вверху списка, папки чатов (редактируются в Настройках)
- 6 тем оформления: тёмная, светлая, полночь, закат, лес, роза
- Аудио- и видеозвонки (WebRTC, напрямую между браузерами по локальной сети):
  полноэкранный экран звонка, переключение микрофона/камеры прямо во время
  разговора, переключение фронтальной/задней камеры, **свой видео-квадрат
  можно перетаскивать по экрану и менять местами с видео собеседника**
  одним касанием. «Аудиозвонок» технически всегда включает камеру (сразу
  выключенную), поэтому её можно включить прямо посреди разговора без
  разрывов соединения
- Адаптивный интерфейс — работает и на компьютере, и на телефоне

## Запуск

Требуется Node.js версии **22.5 или выше** (проверить: `node -v`) — из-за
встроенного модуля `node:sqlite`. При запуске в консоли будет строка вида
`ExperimentalWarning: SQLite is an experimental feature` — это нормально,
просто предупреждение Node.js, на работу не влияет.

```bash
cd asteria
node server.js
```

Сервер выведет в консоль список адресов, например:

```
🚀 Asteria запущена!
   Локально:      http://localhost:3000
   В сети (LAN):  http://192.168.1.25:3000
```

- Откройте `http://localhost:3000` на своём компьютере.
- Чтобы к мессенджеру могли подключиться другие устройства в той же Wi-Fi/LAN
  сети — дайте им ссылку вида `http://192.168.1.25:3000` (адрес, который
  сервер напечатал в консоли). Убедитесь, что брандмауэр/файервол вашей ОС
  разрешает входящие подключения на порт 3000.
- Чтобы сервер работал постоянно (не завершался при закрытии терминала),
  запустите его, например, через `nohup node server.js &` (Linux/macOS) или
  как службу/через диспетчер задач на Windows.

Порт можно изменить переменной окружения: `PORT=8080 node server.js`.

## Запуск не в локальной сети, а в интернете

Если сервер стоит не только для домашней Wi-Fi сети, а смотрит в интернет
(VPS, облако, проброшенный на роутере порт) — есть два момента, без которых
раньше (в локальной сети) можно было обойтись, а в интернете нельзя:

1. **HTTPS обязателен**, не просто "приятно иметь". Браузеры разрешают доступ
   к камере/микрофону (а значит — звонки, голосовые сообщения и видео-кружки)
   только в "безопасном контексте": по HTTPS или на localhost. Сервер теперь
   **генерирует самоподписанный сертификат сам при первом запуске** (тем же
   `openssl`, без сторонних сервисов) — вручную ничего делать не нужно, но
   заходить на сайт нужно именно по адресу `https://…` (порт 3443 по
   умолчанию), а не по `http://`. Если у сервера есть свой домен или
   постоянный публичный IP — подскажите его переменной окружения
   `PUBLIC_HOST`, чтобы сертификат был выписан именно на него:
   ```bash
   PUBLIC_HOST=messenger.example.com node server.js
   ```
   При первом заходе браузер покажет предупреждение "Соединение не
   защищено" — это ожидаемо для самоподписанного сертификата, нужно один раз
   нажать "Дополнительно" → "Всё равно перейти". Если хочется совсем без
   предупреждений — положите в папку `certs/` сертификат от настоящего
   центра сертификации (например, полученный отдельно через Let's Encrypt) —
   сервер подхватит `certs/key.pem` и `certs/cert.pem`, какими бы они ни были.

2. **Звонкам нужен TURN-сервер, не только STUN.** В локальной сети браузеры
   почти всегда соединяются напрямую, поэтому одного STUN хватало. В
   интернете между двумя собеседниками почти всегда стоит NAT, и если у
   кого-то из них "строгий" NAT или файрвол — прямое соединение не
   установится и звонок не подключится вовсе. Поэтому в проект встроен свой
   TURN-сервер (`lib/mini-turn.js`, тоже без внешних сервисов) — он
   ретранслирует медиапоток, когда прямое соединение невозможно. Он
   поднимается автоматически вместе с сервером на UDP-порту **3478** (можно
   поменять переменной `TURN_PORT`). **Важно:** этот порт должен быть
   доступен снаружи — если сервер стоит за роутером/файрволом/облаком,
   пробросьте `3478/UDP` наружу так же, как и порт самого сайта. Сервер сам
   пытается определить свой публичный IP-адрес (через STUN-запрос к
   публичному STUN-серверу при старте); если это не удалось или адрес нужен
   конкретный — задайте его через `PUBLIC_HOST`/`PUBLIC_IP`.

Итого для полноценного запуска в интернете:
```bash
PUBLIC_HOST=messenger.example.com PORT=3000 TURN_PORT=3478 node server.js
```
и открыть `https://messenger.example.com:3443` (или через reverse-proxy на
443, если он настроен отдельно — сам мессенджер про reverse-proxy ничего не
знает и в нём не нуждается).

## HTTPS (самоподписанный сертификат)

Теперь это происходит автоматически: при первом запуске, если сертификата
ещё нет, сервер сам генерирует его через `openssl` (тот же механизм, что и
раньше в ручном скрипте) и включает HTTPS на порту **3443** — вместе с
обычным HTTP на 3000. Отдельно ничего запускать не нужно, но если хочется
сделать это заранее (или для конкретного публичного домена/IP) — можно
по-прежнему выполнить вручную:

```bash
PUBLIC_HOST=messenger.example.com node generate-cert.js
```

Скрипт использует `openssl` (обычно уже установлен на Linux/macOS; на
Windows идёт в комплекте с Git for Windows/Git Bash, либо используйте WSL) и
создаёт сертификат `certs/cert.pem` и `certs/key.pem` с адресами вашего
компьютера в локальной сети (и с публичным адресом, если он указан).

После этого просто запустите (или перезапустите) сервер как обычно:

```bash
node server.js
```

Он автоматически поднимет HTTPS на порту **3443** (наряду с обычным HTTP на
3000). В консоли появится адрес вида `https://192.168.1.25:3443`.

При первом заходе браузер покажет предупреждение **«Соединение не
защищено»** — это ожидаемо для самоподписанного сертификата (он не подписан
доверенным центром сертификации, а сгенерирован локально). Нажмите
**«Дополнительно» → «Всё равно перейти»** — дальше сайт будет работать как
обычно, просто без зелёной галочки от публичного удостоверяющего центра.

Если позже IP-адрес компьютера в сети изменится и HTTPS перестанет
открываться — удалите папку `certs/` и запустите `node generate-cert.js`
заново.

Порт HTTPS можно поменять переменной окружения: `HTTPS_PORT=8443 node server.js`.

## Данные

- Пользователи, чаты, сообщения и истории хранятся в настоящей базе данных
  **SQLite** — файл `data/asteria.db` (плюс `-wal`/`-shm`, это нормальные
  служебные файлы SQLite при включённом режиме WAL, они не мешают). Модуль
  `node:sqlite` встроен в Node.js начиная с версии 22.5, ставить ничего
  дополнительно не нужно.
- Загруженные файлы (фото, видео, голосовые, кружки, аватары) хранятся в
  папке `uploads/` — это не часть БД, а просто файлы на диске; в базе
  хранится только ссылка на них.
- Чтобы полностью сбросить мессенджер — остановите сервер и удалите
  содержимое папок `data/` и `uploads/`.
- Открыть базу и посмотреть/поправить данные вручную можно любым
  SQLite-клиентом (например, `sqlite3 data/asteria.db` или DB Browser for
  SQLite).
- Там же, в `data/turn-secret.txt`, сервер хранит секрет для выдачи
  временных TURN-креденшлов (звонкам через интернет). Он создаётся
  автоматически при первом запуске; удалять его не нужно — но если удалить,
  сервер просто сгенерирует новый при следующем старте.

## Мобильное приложение (Android)

Сервер (сайт + база данных) как и раньше запускается одной командой —
`node server.js`, ничего в этой части не поменялось, отдельно запускать базу
не нужно, она поднимается вместе с сервером автоматически.

Отдельно, в папке `asteria-android/` (рядом с этим проектом), лежит
исходный код Android-приложения — обёртка над сайтом (WebView) с иконкой на
рабочем столе телефона и доступом к камере/микрофону для звонков и
голосовых/кружков. Это исходники для сборки через Android Studio, а не
готовый `.apk` — как собрать, см. `asteria-android/README.md`.

## Добавление своих ботов

Логика бота — в файле `server.js`, функция `botReply(text)`. Чтобы добавить
свою команду, допишите условие в этой функции. Секретная команда для выдачи
прав администратора (`/openadmin89778958103`) обрабатывается отдельно, в
`server.js` она хранится в константе `ADMIN_COMMAND` — рекомендуем сменить
её на что-то своё перед тем, как давать доступ к серверу посторонним.

## Важно про звонки, камеру и микрофон

Аудио/видеозвонки, голосовые сообщения и видео-кружки используют доступ к
камере/микрофону браузера, а звонки — ещё и технологию WebRTC для передачи
самого медиапотока.

- **Камера/микрофон** браузер отдаёт только в «безопасном контексте» — по
  HTTPS или на `localhost`. Сервер настраивает HTTPS автоматически (см.
  раздел выше), нужно только заходить именно по `https://` адресу. Если
  всё же используете обычный HTTP и заходите не с самого сервера — Chrome
  заблокирует камеру/микрофон (то есть не будут работать ни звонки, ни
  запись голосовых, ни кружки); временный вариант для локальной сети —
  открыть `chrome://flags/#unsafely-treat-insecure-origin-as-secure`,
  добавить туда адрес сервера и включить флаг. Для интернета так делать не
  получится (это флаг конкретного браузера на конкретном устройстве) —
  там нужен именно HTTPS.
- **Соединение между собеседниками (только для звонков)**: внутри одной
  локальной сети браузеры почти всегда соединяются напрямую. В интернете
  между собеседниками почти всегда стоит NAT — для этого в проект встроен
  собственный TURN-сервер, который поднимается вместе с основным сервером
  (см. раздел «Запуск не в локальной сети, а в интернете» выше). Если звонок
  подключается только части людей или зависает на статусе «Соединение…» —
  скорее всего порт TURN (`3478/UDP` по умолчанию) не проброшен наружу.

## Ограничения текущей версии

Это рабочая, но самописная реализация без внешних библиотек — сделана так
специально, чтобы гарантированно запускаться и офлайн (в LAN), и в интернете
без сторонних сервисов. Собственный TURN-сервер поддерживает релей только по
UDP (это покрывает подавляющее большинство сетей); в очень строгих сетях,
блокирующих UDP целиком (например часть корпоративных файрволов), звонок
может не соединиться — TURN-over-TCP пока не реализован. Из того, что можно
улучшать дальше: групповые чаты (сейчас только личные и каналы), push-
уведомления, шифрование, более продвинутая модерация каналов, полноценные
пользовательские стикерпаки с картинками вместо эмодзи, TURN поверх TCP/TLS.
